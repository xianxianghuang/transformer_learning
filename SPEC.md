# CBOW模型需求规格文档

## 1. 项目概述

**项目名称**: CBOW词向量训练系统

**项目类型**: 深度学习NLP应用

**核心功能**: 实现经典的CBOW (Continuous Bag of Words) 模型，从文本语料中学习词向量表示

**目标用户**: NLP研究人员、深度学习学习者

## 2. 功能需求

### 2.1 数据预处理
- **文本加载**: 从文件加载原始文本数据
- **文本清洗**: 小写化、去除标点符号、过滤空词
- **词汇表构建**: 基于最小词频过滤，生成词到索引的映射
- **训练数据生成**: 生成(context, target)词对用于训练

### 2.2 CBOW模型
- **词嵌入层**: 将词索引映射为指定维度的密集向量
- **上下文处理**: 对窗口内所有上下文词的嵌入取平均
- **负采样优化**: 使用负采样技术加速训练
- **权重初始化**: 使用均匀分布初始化

### 2.3 训练模块
- **小批量训练**: 支持batch_size配置
- **优化器**: 使用SGD优化器
- **学习率调度**: 支持学习率衰减
- **模型保存**: 支持定期保存和最终保存

### 2.4 评估模块
- **词相似度**: 计算并展示与目标词最相似的词
- **词类比**: 执行"a:b :: c:?"类比推理任务
- **可视化**: 支持t-SNE降维可视化

## 3. 技术规格

### 3.1 超参数配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| embedding_dim | 300 | 词向量维度 |
| window_size | 5 | 上下文窗口大小 |
| min_count | 5 | 最小词频 |
| **learning_rate** | **10.0** | 学习率 (重要: 需使用较大值) |
| **epochs** | **100** | 训练轮数 (推荐100以获得更好效果) |
| batch_size | 512 | 批次大小 |
| negative_samples | 5 | 负采样数量 |

### 3.2 数据流

```
原始文本 -> 预处理 -> 词汇表构建 -> 训练数据生成
                                              |
                                              v
词向量可视化 <- 模型评估 <- 损失计算 <- 模型训练
```

### 3.3 模型架构

```
输入层: context词索引 (batch_size, window_size*2)
    ↓
Embedding层: (batch_size, window_size*2, embedding_dim)
    ↓
平均池化: (batch_size, embedding_dim)
    ↓
输出层: 负采样预测
```

## 4. 验收标准

### 4.1 功能验收
- [x] 能够加载并预处理文本数据
- [x] 词汇表正确构建，词频过滤生效
- [x] CBOW模型正确实现前向传播
- [x] 训练脚本能够正常运行
- [x] 评估脚本能够展示词相似度
- [x] 评估脚本能够执行词类比任务

### 4.2 性能验收
- [x] 训练损失能够持续下降
- [x] 词向量能够捕获语义相似性
- [x] 词类比任务能产生有意义的结果

### 4.3 训练效果验证
- 损失曲线: 从约3.8持续下降到2.35左右（PTB语料）
- 词相似度: 相似词排在前列（如金融领域: million-billion=0.70）
- 词类比: 在干净小规模语料上效果完美，大规模语料需要更多训练

### 4.4 已知问题与解决方案

**问题**: 嵌入层初始化太小导致梯度消失
**解决**: 将初始化范围从 `0.5/embedding_dim` 改为 `0.1`

**问题**: 默认学习率0.025太小，训练不收敛
**解决**: 推荐使用学习率10.0
